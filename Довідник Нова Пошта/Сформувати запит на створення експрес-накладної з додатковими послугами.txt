Сформувати запит на створення експрес-накладної з додатковими послугами
Метод save працює в моделі InternetDocument. Цей метод дозволяє створювати/формувати експрес-накладну (інтернет-документ).

Є можливість створювати запити з додатковими послугами.

Формування запиту на створення експрес-накладної (ЕН) із послугою «Суботня доставка»
Приклад:

"SaturdayDelivery": "1" // 1 — так, послуга «Cуботня доставка» надається, 0 — ні, послуга не надається

Формування запиту на створення ЕН із послугою «Контроль оплати»
Приклад:

"AfterpaymentOnGoodsCost": "1005"

Формування запиту на створення ЕН із послугою «Локал Експрес»
Приклад:

"LocalExpress": "1",

"TimeInterval": "CityDeliveryTimeInterval2" 

Доступні часові інтервали: 2, 3, 4. Якщо це значення порожнє, API проставить позначку "Протягом дня" (див. пункт  «Формування запиту на створення ЕН із послугою «Доставка в часові інтервали»). Послуга доступна лише в межах одного міста.

Формування запиту на створення ЕН із послугою «Дата доставки»*
(*послуга надається тільки в разі доставки до адреси)

Приклад:

"PreferredDeliveryDate": "07.03.2021"

Формування запиту на створення ЕН із послугою «Доставка в часові інтервали»*
(*послуга надається тільки в разі доставки до адреси)

Приклад:

"PreferredDeliveryDate": "25.03.2021",

"TimeInterval": "CityDeliveryTimeInterval1/2/3/4"

Послуга «Доставка в часові інтервали» можлива лише за умови замовлення послуги «Бажана дата доставки». Як завантажити список видів часових інтервалів, можна побачити у «Види часових інтервалів».

Формування запиту на створення ЕН за допомогою послуги «Вказати номери паковання»
Приклад:

"PackingNumber": "4545" // де 4545 — номер паковання

Формування запиту на створення ЕН із послугою «Вказати внутрішній номер замовлення клієнта»
Приклад:

"InfoRegClientBarcodes": "121212" // де 121212 — внутрішній номер клієнта

Формування запиту на створення ЕН із описом «Супровідні документи»
Приклад:

"AccompanyingDocuments" : "Договір"

Формування запиту на створення ЕН із описом «Додаткова інформація про відправлення»
Приклад:

"AdditionalInformation": "Смартфон"

Формування запиту на створення ЕН із послугою «До дверей»
Приклад:

"NumberOfFloorsLifting": "3" // де 3 — номер поверху

Формування запиту на створення ЕН із послугою «Від дверей»
Приклад:

"NumberOfFloorsDescent": "3" // де 3 — номер поверху

Параметр «Ліфт» (Elevator) можна ігнорувати.

Формування запиту на створення ЕН із додатковою послугою «Доставка особисто в руки»
Приклад:

"DeliveryByHand": "1" // 1 — так, послуга надається, 0 — ні, послуга не надається 

"DeliveryByHandRecipients": [  "Могилевська Юлія Костянтинівна", "Колесник Костянтин Олександрович" ] 

У значенні DeliveryByHandRecipients зазначається перелік осіб, яким можна віддати відправлення. Під час замовлення послуги можна завантажувати до 15 осіб включно. Перша особа завжди — одержувач (підтягується з «Дані одержувача»), решта 14  — довільні особи.

Формування запиту на створення ЕН із послугою «Контроль поштучної передачі» (експедирування)
Приклад:

"ForwardingCount":"126" // де 126 — кількість штук

Формування запиту на створення ЕН із послугою «Red Box»
Приклад:

"RedBoxBarcode":"0105QD26L" // код враховує лише верхній регістр

Формування запиту на створення ЕН із пакуванням
Доступно для технології доставки до відділення. Вибрати пакування необхідно за допомогою методу getPackListSpecial. 

Приклад:

{ "modelName": "Common",

"calledMethod": "getPackListSpecial",

"methodProperties": { "Length": 100, // довжина відправлення, мм

"Width": 500, // ширина відправлення, мм

"Height": 50, // висота відправлення, мм

"PackForSale": "1" } }

Ідентифікатор пакування, що повертається у відповіді, необхідно передавати в масиві OptionsSeat. Приклад:

"OptionsSeat": [{ // масив із даними

"weight": 5, // вага відправлення, кг

"volumetricWidth": 80, // ширина відправлення, см

"volumetricLength": 25, // довжина відправлення, см

"volumetricHeight": 25, // висота відправлення, см

"packRef": "1499fa4a-d26e-11e1-95e4-0026b97ed48a" }],

Формування запиту на створення ЕН із послугою «Ручне оброблення»
Послуга доступна до замовлення у разі створення документа з характеристиками: довжина відправлення (або одного місця) — від 1 до 120 см, ширина та висота — до 70 см та вагою (фактичною або об'ємною) відправлення (або одного місця) — від 0,1 до 30 кг. Оформлення відправлень з категорією «Ручне оброблення» дозволено лише з/на вантажні відділення, пункти прийому-видачі та адресу. Замовлення послуги можливе лише з передачею габаритів місць з використанням параметра specialCargo:

"specialCargo":"1" // 1 — так, послугу замовлено, 0 — ні, послугу не замовлено

"OptionsSeat": [ { // масив із даними параметрів вантажу для кожного місця відправлення

"weight": 5, // вага місця, кг

"volumetricHeight": 50, // висота місця, см

"volumetricWidth": 10, // ширина місця, см

"volumetricLength": 10, // довжина місця, см

"cost": "1", // оціночна вартість місця, грн

"description": "1", "specialCargo":"1" }]

Після створення інтернет-документа у програмному середовищі АРІ, ЕН з'явиться в переліку ЕН у бізнес-кабінеті. Якщо відкрити документ, можна побачити послугу та вибране пакування.
