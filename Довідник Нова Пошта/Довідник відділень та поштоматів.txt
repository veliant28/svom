Довідник відділень та поштоматів
Метод getWarehouses працює в моделі Address. Цей метод завантажує довідник відділень компанії «Нова пошта» у контексті населених пунктів України.

Рекомендований формат запиту — POST. Запит повертає HTTP 303 Redirect із посиланням на кешований файл, що містить актуальні дані.
Для коректного отримання даних необхідно враховувати, що для різних міст можуть використовуватися різні типи відповіді. У деяких випадках дані доступні за редіректом на кешований файл, в інших — повертаються безпосередньо. Список міст, для яких працює редірект, може змінюватися. Тому рекомендовано динамічно обробляти відповіді API.
Оскільки кількість записів у відповіді обмежена (не більше 6500), рекомендується виконувати окремі запити для поштових та вантажних відділень, а також поштоматів. Для цього скористайтеся фільтром TypeOfWarehouseRef — тип відділення. Також довідник можна завантажувати посторінково. Для цього виконується запит у циклі з використанням параметрів Page та Limit. Умова завершення циклу — порожній блок даних.
Регулярне щоденне оновлення довідника є необхідним для того, щоб користувачі мали доступ до актуальних даних про всі наявні відділення та поштомати. Наприклад, якщо поруч із клієнтом з'явиться новий поштомат, а ваш довідник не оновлений, клієнт не зможе вибрати його для доставки. 

Рекомендації для забезпечення відображення актуальних даних про відділення та поштомати
Налаштуйте регулярне оновлення довідника. Рекомендовано налаштувати регламентне оновлення довідника всіх складів, відділень та поштоматів щоночі для забезпечення актуальності інформації.
Протестуйте коректність роботи оновлення. Після впровадження функції оновлення  слід переконатися, що пошук на сайті відображає всі відділення, включно з пошуком за містами та типом складу.
Реалізуйте ручний пошук. Для зручності користувачів рекомендовано використовувати фільтрацію за типом та номером відділення. Це дозволить клієнтам швидко знаходити необхідні пункти доставки та відправки.
Є можливість знайти необхідні відділення за такими значеннями:

TypeOfWarehouseRef — тип відділення 
PostFinance — наявність каси NovaPay 
BicycleParking — наявність паркування 
POSTerminal — наявність пос-термінала 
CityName — пошук за назвою населеного пункту 
CityRef — пошук за ідентифікатором населеного пункту
CityName та FindByString — пошук конкретного відділення у вибраному населеному пункті або ж за ідентифікатором відділення
WarehouseId — пошук відділення за його номером
SettlementRef — для отримання відділень у конкретному населеному пункті (значення SettlementRef відображається у відповіді запиту getSettlements)
